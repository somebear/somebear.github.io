#知识点#

  1. map的用法：

    map(function, iterable, …)
    Apply function to every item of iterable and return a list of the results. If additional iterable arguments are passed, function must take that many arguments and is applied to the items from all iterables in parallel. If one iterable is shorter than another it is assumed to be extended with None items. If function is None, the identity function is assumed; if there are multiple arguments, map() returns a list consisting of tuples containing the corresponding items from all iterables (a kind of transpose operation). The iterable arguments may be a sequence or any iterable object; the result is always a list.

    map会将第二个和后面的对象同时遍历，各取出一个元素并将他们当做方法的多个参数传给方法。所以方法名（第一个参数）后面可遍历对象的数量要和执行方法的参数个数相同，而且可遍历对象的长度必须相同，返回的结果是由方法返回值组成的列表。

Learn Python The Hard Way

Posted on 2013-05-12   |  

Softtek每年都会让员工填一个叫Personality Evaluation的excel表格，翻译过来也就是自我评估。表格里面会列出工作各个方面来让员工为自己打分，比如与客户的沟通，对项目的贡献等等。全部填完后会有一个计算公式给出总的成绩，如果高于8分(总分10)而且leader批准的话就可以涨工资的哦(虽然只有5% TAT)。其中有一个问题是你下一步在技术方面有什么学习的方向？我回答是：Learn a new program language – Python。印象这么深是因为当初根本就懒得学，所以连着两年写的都是这个=。，=。

说起来想要学Python最开始是因为看了酷壳上的程序员链接技术攻略。上面介绍说程序员应该懂得一门脚本语言，因为脚本语言可以快速的开发出你能用的上的小程序，比较好的两个选择是Python和Ruby。开始时着实在这两个之间犹豫了很久，因为Python语法简洁优雅，而Ruby拥有逼格极高的Ruby-on-Rails在web开发上更有优势，而且我对比了一下的确用Ruby写的网站大多比用Python的逼格高。这就让我陷入了实用性和装13性的选择困境，毕竟装13是人类进步的阶梯嘛。最后还是选择了Python(优雅一直是我的座右铭好嘛)并在京东300减150的活动时怒入4本Python的书：《Python基础教程》、《Python核心编程》、《Python参考手册》和《Python Cookbook》，并让他们在箱子里面成功的躺了2年（Orz）。到后来公司的项目不忙了就突然想起要学它了，首先去Code Cademy学了一下基本语法，其中Python对列表操作的灵活性和Lambda语句的简洁性惊艳到了我这个苦逼Java党。随后去Coursera报了几门python相关的课程并粗略的啃完了《Python基础教程》——大家推荐的Python入门书，浅显易懂并且涉及了大部分日常所需的知识点。现在正在看《Python Cookbook》，里面是一些针对特殊任务的代码片段，我会将他们都自己写一遍并发布到自己的主页来，希望可以督促自己的学习，让Python像Java一样成为自己的主力语言。

在搜索与Python相关的信息的时候发现了这么一个网站learn python the hard way。在里面有么一句话：The Hard Way Is Easier。是的，对于学习来说并没有什么捷径可言，一点一滴的刻苦努力就是最简单的方法。
