iOS 7下的Cannot find an outgoing row head for incoming head错误

发表于 2015-06-16   |   分类于 life

由于公司的App新版本要适配iPhone 6和iPhone 6plus，就用Masonry将很多视图的布局换成了autolayout。由于模拟器和自己的机器都是iOS8，开发时没有出现什么特别难的问题(其实很多坑)。但是在测试时在iOS 7的设备上出现了一个匪夷所思的问题，某个页面在push一个新的页面时发生了崩溃。而这个问题坑了我两天=_=。

错误日志会输出大量的和constraint有关的信息，最后跟着一句
internal error. Cannot find an outgoing row head for incoming head UIView:0x189712b0.Width, which should never happen.的话。should never happen 你妹啊，这么一大坨错误日志简直吓死爹了。看着这么不正经的log，我的第一反应就是Masonry作者你个坑货（巨巨对不起！）。然后就上网查啊，有没有和我一样倒霉的人啊。结果这是一个iOS 7的bug！

Good job Apple, keep going.

SO上有一个讨论这个bug的问题，下面有一个答案分析的比较详尽，大概意思就是可能触发这个crash的情景有两个：

  1. constants设置的有问题，比如view1.left = view2.left + 20，而view2是nil。或者constants乘以了0。

  2. 遇到了auto layout引擎关于浮点数失去精度的bug，遇到这种问题，你只要在输出的海量log里搜索e-，如果能搜到类似于-7.45058e-08这种超小的数，那就说明你中招了。

    3. 而我就遇上了第二种情况，答主说可以通过调整添加constrains的顺序，或许可以避开这个问题，因为引擎计算的方式变了。下面评论的人还有说把乘以的1.6改成了1.59999这种逗比的方式解决了bug的。然而这些解决方法并没有什么卵用，因为log里提示是一个UIImageView出错了，我把那个视图主要的UIImageView删了加，加了删。就为了看到底是那个出了错。最后真的是鬼使神差般的把注意力集中在了一个小小的图片上，这个图片的作用是指示另一个视图的展开收缩状态。那个视图一开始的高度是0，点击之后会有一个逐渐展开的动画，随着动画这个图片会旋转180度。然而有另外一个Label的右边界好死不死的是以这个图片的左边界为约束的，然后估计旋转的时候发生了什么不可知的计算，触发了auto layout引擎的这个bug。之后我把那个label的右边界改成和父视图固定距离（图片的宽度），crash就不见了。


    这个坑实在是太奇葩，特此纪念，以助后人。QAQ

    伴我同行

    发表于 2015-06-06   |   分类于 life


    今天拿着从工爸潜规则的电影票去看了蓝胖子（感谢寡妇）。一直以为这个陪伴广大同学长大的喵型机器人是个铁皮的，而且是打了蜡的（电影里也有这个桥段），大概那时候能见到的比较高级的机器就是车了吧。结果在3D眼镜下呈现的是一个类似于大白一样的柔软而且弹性十足的胖子，然而并不是充气的。

    哆啦A梦是大雄的曾孙从22世纪带来给他的，唯一的任务就是让大雄幸福，从而引起世界线变动（El Psy Congroo！）。如果成功的话，大雄就会进入迎娶白富美静香从而走向人生巅峰的β世界线。失败的话，面对他的就是创业失败和迎娶胖妹，从而导致整个家族一直穷到曾孙那代（娶胖妹毁3代）α世界线。初期哆啦A梦使用大量道具甚至大杀器印随蛋都没有将静香好感度up，甚至让出木杉成功上位。之后大雄领会到了一个道理，只有靠自己赢得静香才是对的。后面就是屌丝凭借善良感动白富美的故事啦。编剧甚至用“我实在是太优秀了，所以输给了你”这种理由强行将出木杉出局，来照顾广大观众的心情，实在是太拼了。

    不过电影整体还是很不错的，3D的建模很自然，成功的将各个角色从二次元进入了三次元。我一度认为小夫的3D化是无法完成的任务，但是制作组不但成功了，还保留了标准特征。还有最后大雄喝了说谎800之后和哆啦A梦的相聚泪点十足，最后的彩蛋也很有趣。

    网虫

    发表于 2014-05-02   |   分类于 开点脑洞

    网虫

    一种以网络信号为食的虫子。身体呈www形，背部有“0”，”1”相间的花纹，最后一位为1的是公虫，为0的是母虫。其尾部有两个用于接收信号的触角，头部有可以发射信号的口器。没有眼睛，通过发送和接收信号感知周围环境。网虫会入侵并控制人类家中的路由器，代替路由器接收和发送网络请求，趁机吞食数据。网虫虫害严重的家庭会经常遇到打不开网页，QQ消息发送失败，看视频1秒1卡，团战放不出大，秒杀小米F码失败等问题。是困扰新世纪人类的几大害虫之一。

    PS：经科学家研究证实，民间流传的“下死你丫的”法(即通过使用迅雷，BT等工具一次性传输大量数据，让网虫吞食过量数据爆炸而亡)的成功率并不高。因为该方法只有趁网虫睡觉的时候突然袭击才有效，网虫醒着的时候会采用“断你丫网法”即停止接收数据来保护自己。然而由于网虫没有眼睛，无法通过睁眼闭眼来判断它是否在睡觉。科学家表示对此无能为力，建议大家重新使用网线。

    OS X连不上wifi

    发表于 2014-03-10

    今天Macbook连不上wifi了，一开始是看不到自己的路由器只能看到其他的人网络，后来能看到了但是连不上去。

    google了一下找到的解决方案：

    重启并在启动时同时按下 command + option + p + r ，启动音响(”dang~”)三次之后即可。

    这个组合键的作用是用来重置NVRAM和PRAM的。

    关于NVRAM和PRAM的内容摘自Apple Support:

    NVRAM/PRAM: Mac 会将某些设置存储在特定的存储区中，即使关机这些设置也不会丢失。在基于 Intel 的 Mac 上，存储位置是称为 NVRAM 的内存；而在基于 PowerPC 的 Mac 上，存储位置则是称为 PRAM 的内存。
    存储在 NVRAM/PRAM 中的信息包括：

    • 扬声器音量

    • 屏幕分辨率

    • 启动磁盘选择

    • 最近的内核崩溃信息（如果有的话）

    如果遇到的问题与上述功能相关，您可能需要重置 NVRAM 或 PRAM。例如，如果 Mac 并非从“启动磁盘”偏好设置中指定的启动磁盘启动，或者启动 Mac 时暂时出现了“问号”图标，重置 NVRAM 或 PRAM 可能会有帮助。

    注：OS X 不会将网络设置存储在 NVRAM/PRAM 中。如果您正在故障诊断网络问题，重置不会有任何帮助。

    虽然官方文档说对网络故障没有任何帮助，但是却确实解决了我的问题。。。

    unwind

    发表于 2014-02-20   |   分类于 iOS

    在iOS中可以利用Segue来完成从A视图到B视图的切换，而若想从B视图返回A视图就可以用Unwind了。

    实现unwind非常简单：

    1 在你要unwind回去的view controller里实现一个以UIStoryboardSegue为参数的action method。（如果想从B视图unwind回A视图，就在A视图中加）

    2 在要执行unwind的组件上ctrl拖拽到自己view controller的Exit按钮上，在弹出框上选择要绑定的方法。

    PS: 如果有多个unwind到同一个view controller的话可以利用传入UIStoryboardSegue的sourceViewController来判断是由哪个view controller返回的。
